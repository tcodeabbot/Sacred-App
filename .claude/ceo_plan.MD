# CEO Plan: Sacred App — Full Feature Expansion
Generated by /plan-ceo-review + /office-hours on 2026-04-23
Branch: main | Mode: SCOPE EXPANSION
Repo: tcodeabbot/Sacred-App

## Problem Statement

Christians with smartphones consistently fail to protect prayer time because their phone's apps are designed to compete for attention. Sacred solves this by physically blocking distracting apps at scheduled prayer times and replacing them with a prayer relay screen. The mechanic is proven (Prayer Lock has 73K+ users with a thin version), but Sacred's opportunity is to be the premium, deep version — a complete prayer ecosystem, not just a blocker.

## Competitive Context

- **Prayer Lock** (73K users): blocks apps, shows verse/prayer. Thin experience.
- **Holy Focus**: blocks apps, shows handpicked prayer + icon. Still thin.
- **Hallow / Glorify**: deep content (guided audio, community, 20M+ users). No blocking.
- **Sacred's gap**: the first app to combine serious blocking with a serious prayer experience.

## Landscape Synthesis

Layer 1 (tried-and-true): prayer reminder + content apps dominate the market.
Layer 2 (new): Christian app blockers launched 2024-2025 and are growing fast.
Layer 3 (first-principles): EUREKA — blockers are thin. Nobody has built the premium version yet. Sacred's moat is the interrupt mechanic + deep prayer experience together.

## Premises (agreed)
1. The app-blocking interrupt mechanic is the right hook.
2. The prayer library + personalization is what makes it faith-specific vs generic focus app.
3. Streaks + Journey stats create the habit loop driving retention.
4. Auth: user is authenticated via Supabase (Google/Facebook). All new data entities are scoped to `user.id`.

## Effort Legend
S = < 1 day | M = 1–3 days | L = 3–7 days | XL = 1+ week

## Phase Plan
- **Phase 1 (ship first):** Home improvements + Intercept improvements + Prayers search + Prayers curated library + Journey heat map + Achievements + Settings themes/denomination/emergency pass + Daily Word screen + Answered Prayers screen
- **Phase 2:** Prayer Journal + Prayer Requests + Guided Sessions + Audio (TTS) + Share Journey card + Accountability partner
- **Phase 3:** Community / Prayer Circle + Widgets + Church group integration

## Data Layer (new entities)
All stored in Supabase. Local-first via AsyncStorage cache.

```
JournalEntry: { id, user_id, session_id (nullable), date, mood_tag, prayer_text, reflection_text, created_at }
PrayerRequest: { id, user_id, person_name, request_text, priority, group ('personal'|'family'|'church'|'world'), prayed_dates[], answered_at (nullable), answer_note, created_at }
AnsweredPrayer: { id, user_id, prayer_request_id, answered_at, note, created_at }
DailyWord: { id, date, scripture_ref, scripture_text, context_note, reflection_q1, reflection_q2, reflection_q3, themed_prayer_text }

// Curated prayer library (static JSON bundled in app, not Supabase)
LibraryPrayer: { id, title, body_text, category ('morning'|'evening'|'gratitude'|'anxiety'|'confession'|'intercession'|'healing'), tags[], denomination_tags[], source, audio_url (nullable) }

// Per-user library prayer state (Supabase, user_id scoped)
UserLibraryPrayer: { id, user_id, library_prayer_id, is_favorite, last_prayed_at (nullable), session_count }

// Achievements (Supabase, user_id scoped)
UserAchievement: { id, user_id, badge_key ('first_prayer'|'streak_7'|'prayers_100'|'hour_with_god'|'morning_warrior'), unlocked_at }
// Unlock check fires on session completion (in completeSession() action in useAppStore).
// Checks: first_prayer = totalPrayers === 1; streak_7 = currentStreak >= 7;
// prayers_100 = totalPrayers >= 100; hour_with_god = totalMinutes >= 60;
// morning_warrior = 7 consecutive sessions completed before 10am.

// Notification history (AsyncStorage only — no Supabase. Key: 'notification_history')
NotificationRecord: { id, type ('missed_prayer'|'answered_prayer'|'daily_word'|'streak_alert'), title, body, received_at, read: boolean }
// Appended on Expo Notifications foreground listener. Max 50 records (FIFO eviction).

// Grace period (AsyncStorage + App Group UserDefaults)
// Stored in both: AsyncStorage for main app reads; App Group 'group.sacred.app' UserDefaults for extension access.
// Key: 'grace_period_seconds'. Written in Settings on change, immediately synced to App Group.

// Emergency pass (App Group UserDefaults + AsyncStorage for UI)
// App Group key: 'emergency_passes_today' (count used), 'emergency_passes_date' (last reset date ISO string)
// Reset at midnight local time (checked on foreground). Extension reads App Group to gate bypass.
```

- `category` on `LibraryPrayer` is the field used for Journey "prayer category breakdown" stat (session inherits category from the prayed LibraryPrayer).
- `denomination_tags[]` is the field used to filter curated library by user's denomination setting (client-side filter, not a server query).
- `last_prayed_at` on `UserLibraryPrayer` drives the "Last prayed X days ago" label on prayer cards. Updated when a session completes with that prayer selected.
- A session record stores `library_prayer_id` (nullable) to link session → library prayer → category.

DailyWord content: static seed file of 365 entries pre-authored. Start simple. CMS migration in phase 3.

## Technical Constraints
- Screen Time / ShieldConfigurationExtension: the iOS intercept screen runs in a sandboxed extension. Duration picker, animated orbs, and haptics must be validated within ShieldConfigurationExtension / ShieldActionExtension capabilities. RN bridges may not work the same as in the main app.
- Heat map: custom SVG component required (no off-the-shelf RN component). react-native-svg.
- Share card: react-native-view-shot to capture a View as an image for the share sheet.
- TTS (phase 2): expo-speech for MVP. Evaluate ElevenLabs for higher-quality voices in v2.
- Ambient audio: royalty-free nature tracks from freesound.org or Pixabay Audio, bundled in app (< 5MB total, mp3 format). Audio library: `expo-av` with audio session category `Playback` + `mixWithOthers` so it doesn't interrupt system audio. Looped via `Audio.Sound.setIsLoopingAsync(true)`. Streamed from CDN in phase 2.
- Liturgical calendar: static lookup table for 2026–2028 (computed once). Multi-tradition support in phase 2.

## FEATURE BRAINSTORM BY SCREEN

### Screen 1: Home (app/(tabs)/index.tsx)

**Current state:** Sacred header, NextPrayer, hero ProgressRing, EncouragementText, StreakCard, Quick Actions (Pray Now / Test Lock / Scripture), Today's Pauses list, FAB for prayer schedule.

**What's good:** The ProgressRing hero is visually strong. NextPrayer is the right top-of-page element. Streak card gives the habit loop.

**What's missing / improvements:**

1. **Personalized greeting** — "Good morning, [name]. Your next prayer is in 23 minutes." Computed from user name + next scheduled prayer time. Warm and specific.

2. **Daily Scripture chip** — A single verse card above the Quick Actions. Tappable to expand with reflection. Rotates daily. Currently there are scriptures in `constants/prayers.ts` — surface one prominently on Home.

3. **Replace "Test Lock" quick action** — "Test Lock" is a dev affordance, not a user feature. Replace with: **"Journal"** (opens today's prayer journal entry) or **"Quiet Time"** (starts a full guided session).

4. **Contextual Pray Now** — "Pray Now" currently always triggers the instagram intercept. Instead, show the user their next scheduled prayer name and offer "Pray [name] Now" so it feels intentional, not generic. If no scheduled prayer is enabled, fall back to a LibraryPrayer selected by time of day: 5am–11am → `category: 'morning'`; 11am–5pm → `category: 'gratitude'`; 5pm–midnight → `category: 'evening'`. Selection within category: highest `session_count` (most familiar to user); if none prayed yet, first item in seed order.

5. **Missed prayer indicator** — If a scheduled prayer time passed without completion, show a gentle badge or card: "You missed your 7am prayer. Pray it now?" with a 1-tap button. This is the retention hook. Definition of "missed": `scheduled_time + grace_period` has elapsed and no `PrayerSession` record exists for that `prayer_schedule_id` on today's date. Checked on app foreground via `useAppStore.checkMissedPrayers()`.

6. **Live quiet time counter** — While a prayer session is in progress, the Home screen should show a persistent "In prayer — 4:23" indicator replacing the hero card. Tapping it returns to the active session. Implementation: store `sessionStartedAt` in Zustand; Home reads elapsed time on mount and via a 10-second `setInterval` (not real-time subscriptions — stale by ≤10 sec is acceptable).

7. **Weekly rhythm summary** — Instead of just showing "Today's Pauses," add a small 7-day dot calendar (Sun-Sat) above the list showing which days this week had any prayers. Visual motivation.

8. **Notification bell is dead** — The bell icon in the header does nothing. Wire it to a simple Notification History list (answered prayer alerts, missed prayer notifications, daily word). Route: `app/(modals)/notification-history.tsx`. Nav call: `router.push('/(modals)/notification-history')` from the bell `TouchableOpacity` in `index.tsx:63`.

### Screen 2: Intercept / Block Screen (app/(modals)/intercept.tsx)

**Current state:** Full-screen gradient overlay, pulsing orb animation, random scripture, "Begin Prayer" and "Not Now" buttons, shows which app was intercepted.

**What's good:** The animation is calming and appropriate. The random scripture adds spiritual context.

**What's missing / improvements:**

1. **Show scheduled prayer name** — If the intercept was triggered by a scheduled prayer time, show "Time for your [Morning Prayer]" not just a generic screen. Makes it feel intentional.

2. **"Not Now" → "Snooze 10 min"** — "Not Now" lets users dismiss forever. Replace with: "Snooze 10 min" (first tap) and then "Skip this prayer" after one snooze. Tracks skips for the Journey screen. Harder to rationalize skipping when it says "Skip" not "Not Now." **iOS sandbox note (high risk):** Snoozing from within a `ShieldActionExtension` requires re-triggering a `DeviceActivitySchedule` with a new start time. This must be coordinated via App Group shared `UserDefaults` between the extension and main app. Validate this flow early — it's the most complex iOS interaction in Phase 1. Snooze duration is not user-configurable in Phase 1 (fixed at 10 min).

3. **App being blocked shown prominently** — Show the blocked app icon with a gentle "taking a break from [Instagram]" message. Personalizes the moment.

4. **Quick prayer options on intercept** — Before going to full session, offer 3 duration options. Nav targets: "1 min breath prayer" → `app/(modals)/guided-session.tsx` with `{ type: 'breath', duration: 60 }`; "5 min prayer" → `app/(modals)/active-prayer.tsx` with `{ timerPreset: 300 }`; "Full session" → `app/(modals)/guided-session.tsx` with no preset (user picks inside). Respects the user's context (maybe they have 60 seconds, not 20 minutes).

5. **Breathing animation with cue** — During the intercept, show a "breathe in / breathe out" animation or text below the orb. Sets the tone before the prayer begins.

6. **Haptic feedback on intercept** — Subtle vibration when the intercept screen appears. Signals "this is intentional" not a crash.

### Screen 3: Active Prayer / Prayer Session (app/(modals)/active-prayer.tsx + app/prayer-session.tsx)

**Current state:** Timer, 3 modes (silent / scripture / written), complete button, prayer text if selected.

**What's good:** Multi-mode approach is smart. Timer gives measurability.

**What's missing / improvements:**

1. **Guided prayer structure** — Add a 4th mode: **"Guided"** that launches Guided Sessions (New Screen 5) pre-configured for ACTS (Adoration, Confession, Thanksgiving, Supplication). The active prayer screen gains an entry point; the implementation lives in `app/(modals)/guided-session.tsx`.

2. **Background ambient sound** — Nature sounds, soft worship music, silence options. During a 5-minute prayer, audio environment matters. Use an audio library with royalty-free nature tracks.

3. **Mid-prayer journal** — During scripture/written mode, add a small text field: "What is God saying to you?" On session end, auto-create a `JournalEntry` linked to this `session_id`. User lands on the journal entry pre-populated with the session's scripture and their note.

4. **Scripture rotation** — In scripture mode, let the user swipe to the next verse without ending the session. "Lectio divina lite" — linger on a verse, then move to another.

5. **Progress milestone feedback** — At the 2-minute mark, soft haptic + "2 minutes with God." At 5 minutes, "5 minutes — God sees your faithfulness." Encourages completion.

6. **Prayer completion celebration** — The transition from active to complete needs a moment: a gentle animation, a "Well done" message with a scripture affirmation. Not a pop-up — a full-screen moment.

3. **Answered prayer prompt** — "Did God answer any of your recent prayers? Mark one as answered." One-tap quick action.
4. **Next prayer preview** — "Your next prayer is [Evening Prayer] at 9pm." Keeps the habit visible.
5. **Share moment** — "Share that you prayed today" — simple card with streak count. Optional, no guilt if skipped.

### Screen 5: Prayers Library (app/(tabs)/prayers.tsx)

**Current state:** Collection filter chips, prayer cards, favorites, long-press actions, "Manage Collections" button, add button.

**What's good:** Filter chips work. The teal accent bar on cards is nice visual design.

**What's missing / improvements:**

1. **Search bar** — The most basic missing feature. Users with 20+ prayers need search.

2. **Prayer categories/tags visible on card** — Show a small tag ("Morning," "Anxiety," "Gratitude") on each card. Currently excerpt only.

3. **Curated prayer library** — A "Browse" tab in the Prayers screen showing pre-written prayers from Sacred's library (currently `scriptures` array is there but not well exposed). Organized by theme: morning, evening, gratitude, anxiety, confession, intercession, healing.

4. **Audio prayer option** — Text-to-speech (TTS) for any prayer. User can listen while eyes closed. This is table stakes for premium prayer apps.

5. **Schedule a prayer directly from library** — From a prayer card, one tap to "Schedule this prayer at 7am." Connects Prayers to Settings without detour.

6. **Import/export** — Let users share prayers with friends via link. Or paste in a prayer from another source.

7. **Last prayed date** — Show "Last prayed 3 days ago" on each card. Source: `UserLibraryPrayer.last_prayed_at`. Updated on session completion. Cards for prayers never prayed show nothing (no "never prayed" label in MVP). Creates gentle accountability.

8. **Prayer detail screen improvements** — The prayer-detail modal should have: full text readable mode, font size control (small/medium/large), background color choice (white/sepia/dark), share button, and schedule button. Reading preferences (font size + background) persisted in AsyncStorage key `prayer_reading_prefs` — not Supabase (device-local UI preference).

### Screen 6: Journey / Stats (app/(tabs)/journey.tsx)

**Current state:** Weekly bar chart, stats grid (streak, time, prayers, apps blocked), longest streak card.

**What's good:** The weekly chart gradient is clean. Stats grid covers the basics.

**What's missing / improvements:**

1. **Calendar heat map (GitHub-style)** — Replace or supplement the bar chart with a 12-week or 3-month calendar heat map. Instantly shows habit consistency at a glance. This is the most shareable visual in any habit app.

2. **Monthly / yearly toggle** — Let users switch between week / month / year views. Yearly view as a heat map is the "Spotify Wrapped" moment.

3. **Prayer category breakdown** — "You prayed most about Gratitude (40%) and Anxiety (30%) this month." Data path: session record carries `library_prayer_id` → join to `LibraryPrayer.category` → aggregate by month. Sessions without a linked library prayer (e.g., manual prayer) contribute to an "Untagged" bucket.

4. **Time-of-day analysis** — "You pray best in the morning (7-9am). Consider adding an evening prayer." Behavioral insight.

5. **Personal milestones / achievements** — Unlockable badges: "First Prayer," "7-Day Streak," "100 Prayers," "1 Hour With God," "Morning Warrior" (7 consecutive morning prayers). Display as a badge shelf. Persisted in `UserAchievement` table. Unlock check fires in `completeSession()` on session completion. Heat map tap → no drill-down in Phase 1 (tapping a day does nothing).

6. **Answered prayers count** — A dedicated "Answered" stat that counts marked-answered prayers. Makes the journey feel meaningful, not just habitual.

7. **"Share My Journey" card** — Generate a shareable card (like Spotify Wrapped or Apple Fitness) with streak, total time, top prayer theme. Viral + accountability.

8. **Prayer partners comparison** (phase 2) — Opt-in streak comparison with one accountability partner.

### Screen 7: Settings (app/(tabs)/settings.tsx)

**Current state:** Pause Settings (apps, duration, goal), Prayer (schedule, style), Account, About.

**What's good:** Clean grouped iOS-style list. Right pattern for settings.

**What's missing / improvements:**

1. **Notification customization** — Which notifications to receive, sound selection (gentle chime vs. silent), notification text style (scripture vs. reminder).

2. **Appearance** — Light / Dark / System / Sepia themes. System-default dark mode is expected by iOS users.

3. **Prayer reminder tone** — The sound played when prayer time arrives. Calming options: church bell, gentle chime, soft harp, silent.

4. **Grace period setting** — "Allow me to finish what I'm doing for up to [X min] before blocking." Default: 0 (no grace). Valid range: 0–5 minutes in 30-second increments. Currently hard-coded at 0.

5. **Denomination / tradition** — Catholic, Protestant, Evangelical, Pentecostal, Non-denominational. Stored in user settings. Affects which prayer templates appear in the library: client-side filter on `LibraryPrayer.denomination_tags[]`. No server-side query change needed.

6. **Prayer style expanded** — Currently a single field. Should be: silent reflection, scripture-guided, spoken prayer, journaling, contemplative. Each changes the default mode in active prayer.

7. **Emergency pass** — "Allow [X] emergency passes per day where I can bypass the block." Default: 2 passes/day. Counter persisted in App Group UserDefaults (`emergency_passes_today`, `emergency_passes_date`). Resets at midnight local time (checked on app foreground and when extension fires). Extension reads App Group to gate bypass. UI shows remaining passes in Settings and on the intercept screen.

8. **Accountability partner** — Add a friend's email. They get a weekly summary of your streak. No social feed, just one trusted person.

9. **Lock screen style** — What the intercept screen looks like: minimal (just the orb), scripture-heavy, or guided (with prompts).

## NEW SCREENS TO ADD

### New Screen 1: Daily Word
**Route:** `app/(tabs)/daily-word.tsx` or integrated into Home

The anchor for non-blocking users. Every morning, one scripture, one reflection question, one themed prayer. Like Glorify's "10 minutes of quiet time" but without the fluff.

Features:
- Today's date + season of the Church year (Advent, Lent, Ordinary Time)
- Scripture passage with context (1-3 verses + 1 sentence why it matters today)
- 3 reflection questions
- 1 themed prayer pre-written for today
- Archive of past daily words (searchable — in-memory filter on `scripture_ref` and `scripture_text` fields from the static seed array; no server query needed)
- Subscribe to push notification for 6am daily delivery
- **Fallback:** If today's date has no entry (e.g., leap day, seed gap), show the previous day's entry with a note "Today's reflection is the same as yesterday's." Seed file covers 365 entries for non-leap years; day 366 on leap years falls back to day 365.

### New Screen 2: Prayer Journal
**Route:** `app/(tabs)/journal.tsx`

The companion to the active prayer session. Write what happened during prayer. See what God has been saying over time.

Features:
- Timeline of journal entries (by date, with prayer session linked)
- Plain text input (MVP; rich text deferred to phase 2)
- Mood/emotion tag per entry (peaceful, grateful, struggling, joyful)
- "What I prayed for" auto-populated from session (empty string if no session linked)
- "What I heard / sensed" free text
- Search across all entries
- Entries private by default, shareable optionally
- **Empty state:** "Your journal is waiting. Complete your first prayer to begin." + "Start a Prayer" CTA button. Manual journal entry (not linked to a session) also allowed from the `+` button.

### New Screen 3: Answered Prayers
**Route:** `app/(modals)/answered-prayers.tsx`. Entry points: (1) Journey screen "Answered" stat tapped; (2) Prayer complete screen "Mark as answered" button; (3) Prayer Requests card swipe/action. Both Prayer Requests and Answered Prayers ship together in Phase 1 — "mark as answered" on a request writes an `AnsweredPrayer` row and pushes to this screen.

The most emotionally powerful feature in prayer apps (Echo Prayer uses this well). When God answers a prayer, marking it creates a tangible record of faithfulness.

Features:
- Mark any prayer request as "answered" with one tap
- Add a note: "How was it answered?"
- Date answered recorded
- "Remembrance" view: chronological list of answered prayers
- Stats: total answered prayers count (% answered deferred — denominator definition is ambiguous)

### New Screen 4: Prayer Requests / Intercession List
**Route:** `app/(tabs)/requests.tsx`

Different from the prayer library (which is written prayers). This is a list of people and situations to pray for.

Features:
- Quick add: "Pray for [person/situation]"
- Prayer request cards: person's name, request, date added, priority
- "Prayed for this today" one-tap mark — appends ISO date string to `prayed_dates[]` (e.g., `["2026-04-23"]`). Idempotent: duplicate dates not appended.
- Reminder: "You haven't prayed for [Mom's health] in 7 days" — scheduled via `Expo.Notifications.scheduleNotificationAsync` with a weekly trigger, set/reset each time the user marks "prayed." Permission: same notification permission already requested for prayer reminders.
- Mark as answered (connects to Answered Prayers screen)
- Group requests: Personal, Family, Church, World

### New Screen 5: Guided Sessions
**Route:** `app/(modals)/guided-session.tsx`

Structured prayer for people who don't know how to pray or want a framework. The biggest gap vs. Hallow.

Session types:
- **ACTS Prayer** (Adoration, Confession, Thanksgiving, Supplication) — 5/10/15 min versions
- **Lectio Divina** — Read, Meditate, Pray, Contemplate
- **Morning Prayer** — Structured daily offering
- **Evening Examen** — Jesuit-style review of the day with God
- **Gratitude Prayer** — 3 things + prayer of thanks
- **Breath Prayer** — 1-minute centering prayer with cadence

Each session is a step-by-step flow with prompts, timer, and optional scripture. Audio guide optional (TTS or recorded).

**Step data structure** (hardcoded JSON in-app at `constants/guided-sessions.ts`):
```
GuidedSession: { type, durationSeconds, steps: GuidedStep[] }
GuidedStep: { index, title, prompt, durationSeconds, scripture_ref (nullable) }
```
Example: ACTS 5-min = 4 steps × 75 sec. Breath prayer = 1 step (60 sec, cadence animation only, no text prompt). Session screen renders steps sequentially; auto-advances on timer unless user taps "Next." Nav params: `{ type: SessionType, durationSeconds?: number }`. If `durationSeconds` is passed from intercept, nearest matching duration variant is selected.

### New Screen 6: Community / Prayer Circle (Phase 2)
**Route:** `app/(tabs)/community.tsx`

The one thing no blocker has. Opt-in accountability and shared prayer.

Features:
- Create or join a "Prayer Circle" (2-8 people)
- Daily check-in: "Did you pray today?" simple yes/no
- Shared streak: Circle's collective days all members prayed
- Group prayer requests (text only, no social feed)
- Weekly digest: "Your circle prayed a combined 37 minutes this week"
- Anonymous option: join without revealing name
- Church group integration: pastor can create a church circle
